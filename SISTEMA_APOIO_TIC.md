# Sistema de Apoio TIC para Provas em Computador

## 📋 Visão Geral

Sistema que permite convocar uma equipa de técnicos/professores de apoio TIC para provas realizadas em computador (Aplicações Informáticas, Design, Multimédia, etc.). Segue o mesmo modelo dos Suplentes, permitindo convocação ad-hoc com número flexível de técnicos.

## ✨ Características

- ✅ **Exame virtual** para apoio TIC (não requer alunos inscritos)
- ✅ **Convocação ad-hoc** - Adicionar/remover técnicos livremente até ao dia da prova
- ✅ **Número indeterminado** - Quantidade flexível conforme necessidade
- ✅ **Alocação de salas** sem restrição de número de alunos
- ✅ **Sistema integrado** - Usa toda a infraestrutura existente de convocatórias
- ✅ **Função dedicada** - 'Apoio TIC' já disponível na tabela convocatoria

## 🎯 Como Usar

### 1️⃣ Criar Sessão de Apoio TIC

1. Aceder a **Sec. Exames → Sessões de Exame**
2. Clicar em **Nova Sessão**
3. No formulário:
   - **Exame**: Selecionar `TIC-APOIO` (Apoio TIC - Provas em Computador)
   - **Fase**: Escolher fase apropriada (ex: 1ªfase, 2ªfase)
   - **Data**: Data da prova em computador
   - **Hora**: 
     - Recomendado: 30-45min antes do início da prova
     - Permite tempo para preparar sistemas e testar equipamento
   - **Duração**: Tempo total da prova + preparação + encerramento (ex: 180-240 minutos)
   - **Tolerância**: 15-30 minutos
   - **Nº Alunos**: ⚠️ **Deixar em branco** (campo desabilitado automaticamente)
   - **Observações**: Ex: "Apoio TIC - Prova de Aplicações Informáticas B - 40 alunos"

4. Guardar sessão

### 2️⃣ Alocar Sala(s) de Computadores

1. Na lista de sessões, clicar em **Alocar Salas**
2. Selecionar sala(s) onde decorrerá a prova:
   - Laboratório de Informática 1
   - Laboratório de Informática 2
   - Sala de Design/Multimédia
3. No campo **Nº de Alunos na Sala**, deixar 0
4. Guardar alocação

⚠️ **Nota**: Para apoio TIC, não há validação de número de alunos

### 3️⃣ Convocar Equipa de Apoio TIC

1. Aceder a **Sec. Exames → Convocatórias/Vigilâncias**
2. Filtrar pela sessão de apoio TIC criada
3. Clicar em **Convocar Vigilantes**
4. Adicionar técnicos/professores:
   - **Função**: Será automaticamente "Apoio TIC"
   - **Quantidade**: Flexível conforme necessidade
     * 1 técnico por sala (mínimo)
     * 1 técnico por 20-25 alunos (recomendado)
5. Enviar notificações

**Pode continuar a adicionar/remover técnicos até ao dia da prova!**

### 4️⃣ No Dia da Prova

#### Pré-Prova (30-45min antes):
1. Equipa TIC marca presença no sistema
2. Preparação dos sistemas:
   - Ligar todos os computadores
   - Verificar funcionamento do software necessário
   - Testar ligação à rede/internet (se aplicável)
   - Criar pastas de trabalho para alunos
   - Configurar restrições de acesso
   - Verificar impressoras/scanners (se necessário)

#### Durante a Prova:
- Disponibilidade para resolver problemas técnicos
- Reiniciar computadores se necessário
- Garantir que não há perda de trabalho dos alunos
- Apoiar vigilantes em questões técnicas
- Documentar incidentes

#### Pós-Prova (15-30min):
- Fazer backup dos trabalhos dos alunos
- Exportar ficheiros para formato requerido
- Verificar integridade dos ficheiros
- Limpar pastas temporárias
- Relatório de incidentes (se houver)

## 🗄️ Estrutura de Base de Dados

### Tabela `exame`
- `tipo_prova` = 'Apoio TIC' (novo valor no ENUM)
- `ano_escolaridade` = NULL para apoio TIC
- Código: `TIC-APOIO`

### Tabela `sessao_exame`
- `num_alunos` = NULL ou 0 para apoio TIC
- Duração típica: 180-300 minutos (inclui preparação + prova + encerramento)

### Tabela `convocatoria`
- `funcao` = 'Apoio TIC' (já existente no ENUM)
- Funciona normalmente com todas as features:
  - Confirmação do professor/técnico
  - Marcação de presença
  - Notificações por email
  - Relatórios e estatísticas

## 📝 Queries Úteis

### Listar sessões de apoio TIC
```sql
SELECT 
    se.id,
    se.data_exame,
    se.hora_exame,
    se.duracao_minutos,
    COUNT(DISTINCT c.id) as num_tecnicos_convocados,
    SUM(CASE WHEN c.presenca = 'Presente' THEN 1 ELSE 0 END) as tecnicos_presentes,
    se.observacoes,
    GROUP_CONCAT(DISTINCT s.codigo_sala SEPARATOR ', ') as salas
FROM sessao_exame se
JOIN exame e ON e.id = se.exame_id
LEFT JOIN convocatoria c ON c.sessao_exame_id = se.id
LEFT JOIN sessao_exame_sala ses ON ses.sessao_exame_id = se.id
LEFT JOIN salas s ON s.id = ses.sala_id
WHERE e.tipo_prova = 'Apoio TIC'
    AND se.data_exame >= CURDATE()
GROUP BY se.id
ORDER BY se.data_exame, se.hora_exame;
```

### Ver equipa TIC convocada para uma data específica
```sql
SELECT 
    c.id,
    u.name as tecnico,
    u.email,
    se.hora_exame,
    se.duracao_minutos,
    c.presenca,
    c.estado_confirmacao,
    c.observacoes,
    GROUP_CONCAT(DISTINCT s.codigo_sala SEPARATOR ', ') as salas
FROM convocatoria c
JOIN user u ON u.id = c.user_id
JOIN sessao_exame se ON se.id = c.sessao_exame_id
JOIN exame e ON e.id = se.exame_id
LEFT JOIN sessao_exame_sala ses ON ses.sessao_exame_id = se.id
LEFT JOIN salas s ON s.id = ses.sala_id
WHERE e.tipo_prova = 'Apoio TIC'
    AND se.data_exame = '2026-06-20'
GROUP BY c.id
ORDER BY se.hora_exame, u.name;
```

### Estatísticas de apoio TIC por técnico
```sql
SELECT 
    u.name as tecnico,
    COUNT(c.id) as total_apoios,
    SUM(CASE WHEN c.presenca = 'Presente' THEN 1 ELSE 0 END) as presencas,
    SUM(CASE WHEN c.presenca = 'Falta' THEN 1 ELSE 0 END) as faltas,
    ROUND(AVG(se.duracao_minutos) / 60, 1) as media_horas_por_sessao
FROM convocatoria c
JOIN user u ON u.id = c.user_id
JOIN sessao_exame se ON se.id = c.sessao_exame_id
JOIN exame e ON e.id = se.exame_id
WHERE e.tipo_prova = 'Apoio TIC'
    AND c.funcao = 'Apoio TIC'
GROUP BY u.id, u.name
ORDER BY total_apoios DESC;
```

### Sessões TIC com equipa reduzida (alerta)
```sql
SELECT 
    se.id,
    se.data_exame,
    se.hora_exame,
    se.observacoes,
    COUNT(DISTINCT ses.id) as num_salas,
    COUNT(DISTINCT c.id) as num_tecnicos,
    CASE 
        WHEN COUNT(DISTINCT c.id) < COUNT(DISTINCT ses.id) THEN 'CRÍTICO - Faltam técnicos'
        WHEN COUNT(DISTINCT c.id) = COUNT(DISTINCT ses.id) THEN 'ATENÇÃO - 1 técnico por sala'
        ELSE 'OK'
    END as status_equipa
FROM sessao_exame se
JOIN exame e ON e.id = se.exame_id
LEFT JOIN sessao_exame_sala ses ON ses.sessao_exame_id = se.id
LEFT JOIN convocatoria c ON c.sessao_exame_id = se.id
WHERE e.tipo_prova = 'Apoio TIC'
    AND se.data_exame >= CURDATE()
GROUP BY se.id
HAVING num_tecnicos < num_salas
ORDER BY se.data_exame, se.hora_exame;
```

### Incidentes técnicos registados
```sql
SELECT 
    se.data_exame,
    se.hora_exame,
    u.name as tecnico,
    c.observacoes as incidentes
FROM convocatoria c
JOIN user u ON u.id = c.user_id
JOIN sessao_exame se ON se.id = c.sessao_exame_id
JOIN exame e ON e.id = se.exame_id
WHERE e.tipo_prova = 'Apoio TIC'
    AND c.observacoes IS NOT NULL
    AND c.observacoes != ''
ORDER BY se.data_exame DESC, se.hora_exame DESC;
```

## 🔄 Rollback

Se necessário reverter as alterações:

```sql
-- Remover exames de apoio TIC
DELETE FROM exame WHERE tipo_prova = 'Apoio TIC';

-- Restaurar ENUM (remover 'Apoio TIC')
ALTER TABLE exame 
MODIFY COLUMN tipo_prova ENUM(
    'Exame Nacional', 
    'Prova Final', 
    'MODa', 
    'Suplentes',
    'Verificacao Calculadoras'
) NOT NULL;
```

⚠️ **Atenção**: O rollback apaga todas as sessões e convocatórias associadas devido ao `ON DELETE CASCADE`.

## 📐 Boas Práticas

### Quantos Técnicos Convocar?

**Regra Base:**
- **Mínimo**: 1 técnico por sala de computadores
- **Recomendado**: 1 técnico por 20-25 alunos
- **Para provas críticas**: +1 supervisor adicional

**Cálculo:**
```
Nº Técnicos = MAX(
    Nº de Salas,
    CEILING(Nº Alunos / 25)
)
```

**Exemplos:**

| Alunos | Salas | Mínimo | Recomendado |
|--------|-------|--------|-------------|
| 20     | 1     | 1      | 1-2         |
| 40     | 2     | 2      | 2-3         |
| 80     | 3     | 3      | 4-5         |
| 100    | 4     | 4      | 5-6         |

### Timing Recomendado

- ⚠️ **30min antes**: Mínimo aceitável

**Durante Prova:**

**Pós-Prova:**

## 🚨 Gestão de Incidentes

### Problemas Comuns e Soluções:

**1. Computador não liga**
- Verificar cabo de alimentação
- Trocar aluno para PC de reserva
- Registar incidente no sistema

**2. Software não abre**
- Verificar licença
- Reinstalar aplicação
- Usar computador alternativo

**3. Perda de ligação à internet**
- Verificar cabo de rede
- Resetar router
- Usar dados guardados localmente

**4. Trabalho não guarda**
- Verificar permissões da pasta
- Guardar em localização alternativa
- Fazer backup manual imediato

**5. Impressora offline**
- Verificar ligação
- Reiniciar serviço de impressão
- Imprimir noutra impressora

### Protocolo de Emergência:

1. **Problema Individual** (1 aluno):
   - Resolver localmente
   - Registar no sistema
   - Informar vigilante

2. **Problema Parcial** (2-5 alunos):
   - Priorizar solução rápida
   - Redistribuir alunos se necessário
   - Alertar coordenador

3. **Problema Global** (>5 alunos ou sistema):
   - Alertar coordenador imediatamente
   - Avaliar necessidade de pausa
   - Documentar extensivamente
   - Contactar suporte externo se necessário
